CSS Selectors and Specificity

What are CSS Selectors?

CSS selectors are interesting to learn because in order to style a bunch of elements inside the HTML document we have to be ready to select and separate them from the rest of the elements which we do not want to style.

There is a whole list of CSS selectors designed to make interesting and exctiting things, but I will test tome right now, below.

How many types of selectors are there?

Well, quite a lot, but let's start slow and show everything first.

  1. Universal and element selector

    This type of selector is interesting because it selects every possible element that is within the HTML code.

  2. The ID selector

    This type of selector is also intresting because it selects something with the given ID.

  3. The Class Selector

    This type of selector makes a styling based on the class selection of those elements

  4. Descendant Selector

    This type of selector selects only the elements that are nested inside another element.

  5. Combinator Selector

    This type of selector selects only the elements that are preceded by another entity.

  6. Direct Child Selector

    This type of selector selects only the elements that are Direct Children by another entity.

  7. Attribute Selector

    This type of selector makes a selection with the elements that have a specific attribute.

The different types of CSS elements

There are around 2 different types of different elements

  • Pseudo Elements
  • Pseudo Classes

  1. Pseudo Classes

    These type of classes have a particular keyword that represent a particular state of that selected element, such as: checkboxes, hovered over, first child etc.

  2. Pseudo elements

    These type of elements have a particular keyword that represents a particular part of that selected element such as: first letter of paragraphs, first line, before and after a paragraph etc

The CSS Cascade Model

Because CSS is an acronym from Cascade Style Sheets, the order in which those elements are styled actually matters a whole lot, whether we have one stylesheet or not

CSS Specificity

There is a general rule that every browser follows, which is focused more around how specific a selection is. The more specific, the more important.

The general formula follows is as follows :

ID > CLASS > ELEMENT

This sorta of war applies only when there is a conflict between two selectors. If those two are equal in specificity, because of the CSS Cascade Model, the one which is a the bottom wins.
